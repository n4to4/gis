# ページ作成ルール

## 1ページ = 1概念

各ページは単一の概念・トピックを扱う。複数の概念をまとめない。

**OK：** `crs.md`（座標系とは何か）  
**NG：** `data.md`（ベクタ・ラスタ・フォーマットをまとめて説明）

## ファイル配置

```
src/content/docs/
├── index.mdx          # 全体の索引（トップページ）
├── <slug>.md          # トップレベルの概念
└── <category>/
    └── <slug>.md      # カテゴリ内の概念
```

- スラッグは英語・小文字・ハイフン区切り（例：`vector-data.md`）
- カテゴリは概念が3つ以上まとまるときに作る

## フロントマター（必須）

```yaml
---
title: （日本語タイトル）
description: （1〜2文の説明）
---
```

## 索引への追記

ページを追加したら `src/content/docs/index.mdx` の対応セクションにリンクを追加する。

```md
→ [ページタイトル](/gis/<slug>/)
```
