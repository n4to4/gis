GISで扱うデータのフォーマット一覧。ベクター・ラスター・タイル・3Dモデルの観点で整理する。

## フォーマット一覧

| フォーマット | 種別 | 概要 |
|---|---|---|
| [CityGML](/gis/data-formats/citygml/) | ベクター | 3D都市モデルを表現するXMLベースのフォーマット。建物や道路などの都市要素を階層的に扱える |
| [GeoJSON](/gis/data-formats/geojson/)（準備中） | ベクター | JSONベースの汎用フォーマット。ブラウザでの利用に向いている |
| [Shapefile](/gis/data-formats/shapefile/)（準備中） | ベクター | ESRI製の古くからある標準フォーマット。複数ファイルで構成される |
| [GeoTIFF](/gis/data-formats/geotiff/)（準備中） | ラスター | 地理情報（座標・CRS）を埋め込んだTIFF画像 |
| [MVT](/gis/data-formats/mvt/) | タイル | Mapbox Vector Tile。ベクター地物をタイル単位で効率よく配信する形式 |
| [PMTiles](/gis/data-formats/pmtiles/)（準備中） | タイル | 単一ファイルで配信できるマップタイル形式 |
| [3D Tiles](/gis/data-formats/3d-tiles/) | タイル | 3D空間データを階層化して段階的に配信・描画するための形式 |
| [glTF](/gis/data-formats/gltf/) | 3Dモデル | 3Dモデルの形状、材質、テクスチャを効率よく表現する形式 |

## 種別について

- **ベクター**：点・線・ポリゴンで地物を表現。属性情報を持てる
- **ラスター**：グリッド状のセルに値を格納。衛星画像や標高データに使われる
- **タイル**：地図を一定サイズの画像や数値に分割したもの。表示の高速化に使われる
- **3Dモデル**：メッシュやマテリアルを用いて立体形状を表現する。可視化や配信形式と組み合わせて使われる
